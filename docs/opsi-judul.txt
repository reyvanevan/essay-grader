### ✅ ALASAN BISA DIANGKAT (The "Why")

1.  **Relevansi Teknologi (State of the Art):**
    Lu pake **LLM (Large Language Models)** dan **Computer Vision** (buat baca tulisan tangan). Ini teknologi paling *hype* dan seksi di mata akademisi sekarang.
2.  **Novelty (Kebaruan) - The Low-Code Angle:**
    Biasanya orang bikin sistem AI pake Python ribet. Lu pake pendekatan **Low-Code Orchestration (n8n)**.
    *Argumen Lu ke Dosen:* "Pak, saya meneliti efisiensi pengembangan sistem cerdas menggunakan metode Low-Code. Jadi development-nya 10x lebih cepet."
3.  **Problem Solving Nyata:**
    Masalah dosen capek ngoreksi itu masalah abadi. Solusi lu menawarkan efisiensi waktu yang terukur.

### ❌ ALASAN GABISA DIANGKAT (The Risk)
*Hanya akan ditolak kalau:*
1.  **Terlalu Simpel:** Kalau lu cuma bikin form yang nembak API ChatGPT mentah-mentah tanpa ada *pre-processing* atau *post-processing*. (Makanya logic "Plagiat Checker" & "Quality Gate" tadi penting).
2.  **Gak Ada Pengujian:** Skripsi butuh data. Lu harus buktiin akurasi AI-nya dibanding koreksi manual manusia.

### 🛠️ APA AJA YANG LU BUTUHIN? (Starter Pack Skripsi)

Biar lu gak buta arah, siapin ini:

**1. Tools (Lu Udah Punya Semua):**
* **Backend Logic:** n8n (Self-hosted di DigitalOcean lu).
* **AI Engine:** OpenAI API (GPT-4o) atau Google Gemini API (Gratis).
* **Database:** TiDB Cloud (MySQL) buat simpen nilai & log plagiat.
* **Frontend:** Next.js (Biar keliatan modern pas demo sidang).

**2. Data Sampel (Wajib Buat Bab 4):**
* Cari 20-30 contoh soal essay/tugas.
* Minta temen lu (atau tulis sendiri) jawaban yang bervariasi:
    * Jawaban Benar Sempurna.
    * Jawaban Salah Total.
    * Jawaban Tulisan Cakar Ayam.
    * Jawaban Copas (Plagiat).

**3. Metode Pengujian (Biar Lulus):**
* **Confusion Matrix:** Bandingin nilai yang dikasih Dosen Manusia vs Nilai AI.
    * *Contoh:* Dosen kasih nilai 80, AI kasih 82. Berarti akurasi tinggi.
    * *Contoh:* Dosen kasih 50, AI kasih 90. Berarti error, analisa kenapa error (mungkin tulisan gak kebaca).
